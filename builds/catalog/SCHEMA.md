# Option Catalog Schema (v2)

**Audience: table-facing** (catalog data is player-visible; no GM secrets).
**Purpose:** the *option catalog* the rung-3 builder validates picks against. Where the
ledger says "I picked Nimble (cost 2)" and the engine checks the *arithmetic*, the catalog
is the thing that knows Nimble **exists**, is an **Elf** trait, and **costs 2** — the legality
layer the engine did not have (RUNG3_PLAN §10 build-item 1, §11 spike 2).

v1 was build-order **step 2** (RUNG3_PLAN §7): the Spellblade pilot. v2 is **step 4**: the
fan-out to all five walked classes (Spellblade, Warlock, Commander, Barbarian, Druid) and the
cross-cutting lists the other five PCs touch. It stays "validate-don't-enumerate": we encode
only the options the walked builds actually touch, plus a level or two ahead, verified by the
66/66 oracle (the six ledgers passing the engine). Content nobody has picked yet is left for
the day someone walks that path.

## Design principles

1. **Spine is scripted; cross-cutting cost lists are hand-curated.** Each class *spine*
   (per-level resource deltas + feature labels, Disciplines / Pact Boons, Subclasses,
   spellcasting model) is generated by `tools/catalog_build.py` straight from the authored
   `class_spines.yaml` + `rules/classes.md` (the engine reads the same data), so it can never drift. The
   *cross-cutting* lists that cut across classes — spell-school and spell-source membership,
   ancestry-trait costs, maneuver names, talent names — are hand-curated YAML, each
   cross-checked against `rules/*.md` by `tools/catalog_verify.py`. (Resolution of the §8
   open catalog decision.)
2. **Additive, never destructive.** A new class or ancestry is new rows, never edits to existing
   ones (the homebrew-overlay principle from `ROADMAP.md`). Same for options.
3. **Data-driven, class-agnostic.** Options are catalog rows, never `if class == …` branches. The
   number six is nowhere structural. More classes later = more data, not an engine rewrite.
4. **Verified by the oracle, not by completeness.** The catalog does not need to be complete. It
   needs the paths the six PCs walk to reconcile with the ledgers the engine already passes 66/66.

## Files

- `builds/catalog/SCHEMA.md` — this file.
- **SCRIPTED spines** (do not hand-edit; regenerate with `python3 tools/catalog_build.py`):
  `spellblade.yaml`, `warlock.yaml`, `commander.yaml`, `barbarian.yaml`, `druid.yaml`.
  Per-level table from `class_spines.yaml`, class L1 choices (Spellblade Disciplines with resource
  `grants`; Warlock Pact Boons), the 3 Subclasses (+ `subclass_grants`, e.g. Eldritch
  Psychic-tag spell access), and the spellcasting model.
- `builds/catalog/spell_schools.yaml` — **CURATED**. School membership for the SCHOOLS-model
  classes (Spellblade: 2 chosen schools + Weapon/Ward tag access; Warlock: 3 chosen schools).
  From the flattened "Spells sorted by Schools" list in `spells.md`. Curated schools:
  Invocation, Divination, Elemental, Nullification, Transmutation.
- `builds/catalog/spell_sources.yaml` — **CURATED**. Source membership for the SOURCE-model
  classes (Druid: any Primal spell). Tracked **source → school → spells**, mirroring the
  "Spells sorted by Sources" section with its parent-source headings — this is where the §11
  same-school-name-under-different-sources wrinkle lives. Curated: Primal (add Arcane/Divine
  when a Cleric/Sorcerer/Wizard ledger arrives).
- `builds/catalog/ancestries.yaml` — **CURATED**. Ancestry-trait names and point costs for the
  nine trait lists the builds walk: Human, Elf, Dwarf, Halfling, Giantborn, Dragonborn,
  Fiendborn, Angelborn (reachable via the Redeemed Fiendborn trait), Beastborn (incl. the
  Beast Traits list). Supports `source_aliases` (ledger "Beast" → "Beastborn") and per-trait
  `aliases` (ledger "Arcane Spell" → rules "Fiendish Magic").
- `builds/catalog/maneuvers.yaml` — **CURATED**. The 0.10.5 maneuver names by type (Attack /
  Defense / Grapple / Utility), from `combat.md`. Deliberately absent: "Recovery" (Bonan's
  sheet), which does not exist in 0.10.5 — the verifier whitelists that ledger placeholder.
- `builds/catalog/talents.yaml` — **CURATED**. General talents, Multiclass talents, class
  talents for the walked classes, and `mc_features` — the class features the six reach via
  Multiclass talents (Pact Spell, Life Tap, Wild Form, Remarkable Repertoire, Innate Power,
  Meta Magic, Spell School Initiate), with their `grants`.
- `builds/catalog/skills_trades.yaml` — **CURATED** (added step 5). The 0.10.5 skill list
  (12, grouped by attribute) and trade list (28, knowledge subset flagged), from
  `core-rules.md`. Drives the builder's skill/trade picker; free-text custom entries stay
  allowed (validate-don't-enumerate — the engine only checks budgets).
- **`opens:` key** (step 5) on cross-list ancestry traits: Redeemed `opens: Angelborn`,
  Fallen `opens: Fiendborn`. In the builder's scratch mode, trait legality = the DECLARED
  ancestry lists plus lists opened by taken traits (to a fixpoint); canon ledgers keep the
  original source-derived resolution.
- `tools/catalog_build.py` — the spine extractor (all five classes).
- `tools/catalog_verify.py` — the oracle harness: re-runs every ledger through the engine, and
  validates the catalog against ALL SIX ledgers and `rules/*.md`.

## Spine files (scripted)

```yaml
catalog_version: 1
ruleset: "DC20 0.10.5"
class: Warlock
generated_by: tools/catalog_build.py     # do NOT hand-edit; regenerate
spine:                                    # per-level, straight from class_spines.yaml
  1: {hp: 8, mp: 6, spells: 4, features: [Class Features]}
  ...
pact_boons_pick_l1: 1                     # class-specific L1 choice block (Spellblade has
pact_boons:                               # disciplines_pick_l1 / disciplines instead)
  - {name: Pact Weapon, grants: {maneuvers: 2}, note: "2 Attack Maneuvers; ..."}
  ...
subclasses: [Eldritch, Fey, Paragon]
subclass_grants:                          # only where a walked subclass confers resources
  Eldritch: {grants: {spells: 1}, spell_access: {tag: Psychic}}
spellcasting: {model: schools, schools_chosen: 3}   # or {model: source, source: Primal}
paths: [Martial, Spellcaster]             # or {model: none, path_rider: "..."} for martials
```

`grants:` mirrors the ledger's `grants:` keys, so a builder can look an option up and know its
resource deltas without the engine special-casing it (e.g. Magus = +1 MP, +1 spell, exactly what
`tanrielle.yaml` records at L3).

## Spell-access models (who reads which list)

- **schools** (Spellblade, Warlock): legal pick = in a CHOSEN school (`spell_schools.yaml`)
  OR carries a tag in `tag_access` (Spellblade Weapon/Ward) OR granted by a subclass
  `spell_access` tag (Eldritch Psychic) OR in a school opened by a talent (Spell School
  Initiate). No source tracking needed.
- **source** (Druid): legal pick = on the class's Source list (`spell_sources.yaml`), plus any
  explicit grant slots (Scaletrix: 2 Arcane spells via Innate Power Intuitive Magic + 1 via
  Fiendish Magic).
- **none** (Commander, Barbarian): no class Spell List; the Spellcaster Path first-time rider
  grants "a Spell List of their choice from any Class" (character-creation.md), and MC Bard
  Magical Secrets is any-list. The choice isn't recorded on the sheets, so the verifier checks
  existence + consistency only.

## `ancestries.yaml` (curated)

```yaml
catalog_version: 1
source_aliases: {Beast: Beastborn}
ancestries:
  Fiendborn:
    - {name: Redeemed, cost: 0, note: "may spend Ancestry Points on Angelborn Traits"}
    - {name: Fiendish Magic, cost: 2, aliases: [Arcane Spell]}
    ...
# cost may be negative (drawback traits refund points). Budget enforcement stays in the engine;
# the catalog only supplies the per-trait cost the engine's arithmetic checks against.
# Lookup order: the trait's stated source list (via source_aliases), then trait aliases, then
# the character's other lists (Runt's unsourced Strong Body -> Giantborn; Scaletrix's Umbral
# Weakness -> Angelborn via Redeemed).
```
