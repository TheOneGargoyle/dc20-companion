# Option Catalog Schema (v1)

**Audience: table-facing** (catalog data is player-visible; no GM secrets).
**Purpose:** the *option catalog* the rung-3 builder validates picks against. Where the
ledger says "I picked Nimble (cost 2)" and the engine checks the *arithmetic*, the catalog
is the thing that knows Nimble **exists**, is an **Elf** trait, and **costs 2** — the legality
layer the engine did not have (RUNG3_PLAN §10 build-item 1, §11 spike 2).

This is build-order **step 2** (RUNG3_PLAN §7): the catalog for the pilot class, Spellblade
(Tanrielle). It is the first slice of the "validate-don't-enumerate" data: we encode only the
options the walked builds actually touch, plus a level or two ahead, verified by the 66/66
oracle (the six ledgers passing the engine). Content nobody has picked yet is left for the day
someone walks that path.

## Design principles

1. **Spine is scripted; cross-cutting cost lists are hand-curated.** The class *spine* (per-level
   resource deltas + feature labels, the Disciplines, the Subclasses) is generated by
   `tools/catalog_build.py` straight from the engine's `CLASS_TABLES` + `rules/classes.md`, so it
   can never drift from the engine. The *cross-cutting* lists that cut across classes — spell-school
   membership and ancestry-trait costs — are hand-curated YAML, each cross-checked against
   `rules/*.md` by `tools/catalog_verify.py`. (Resolution of the §8 open catalog decision.)
2. **Additive, never destructive.** A new class or ancestry is new rows, never edits to existing
   ones (the homebrew-overlay principle from `ROADMAP.md`). Same for options.
3. **Data-driven, class-agnostic.** Options are catalog rows, never `if class == …` branches. The
   number six is nowhere structural. More classes later = more data, not an engine rewrite.
4. **Verified by the oracle, not by completeness.** The catalog does not need to be complete. It
   needs the paths the six PCs walk to reconcile with the ledgers the engine already passes 66/66.

## Files

- `builds/catalog/SCHEMA.md` — this file.
- `builds/catalog/spellblade.yaml` — **SCRIPTED** (do not hand-edit). The Spellblade spine:
  per-level table (from `CLASS_TABLES`), the 8 Disciplines with their resource `grants`, the 3
  Subclasses, the spellcasting model. Regenerate with `python3 tools/catalog_build.py`.
- `builds/catalog/spell_schools.yaml` — **CURATED**. Spell-school membership for the schools the
  builds walk (Invocation, Divination), plus the Spellblade tag-access rule (Weapon / Ward). From
  the "Spells sorted by Schools" list in `spells.md`.
- `builds/catalog/ancestries.yaml` — **CURATED**. Ancestry-trait names and point costs for the
  ancestries the builds walk (Human, Elf). From `ancestries.md`.
- `tools/catalog_build.py` — the spine extractor.
- `tools/catalog_verify.py` — the oracle harness: re-runs every ledger through the engine, and
  validates the catalog against both the ledgers and `rules/*.md`.

## `spellblade.yaml` (scripted spine)

```yaml
catalog_version: 1
ruleset: "DC20 0.10.5"
class: Spellblade
generated_by: tools/catalog_build.py     # do NOT hand-edit; regenerate
spine:                                    # per-level, straight from CLASS_TABLES
  1: {hp: 8, sp: 1, maneuvers: 1, mp: 3, spells: 2, features: [Class Features]}
  ...
disciplines:                              # L1 picks 2; +1 via Paladin Lay on Hands (L3); +1 Expert (L5)
  - {name: Magus, grants: {mp: 1, spells: 1}}
  - {name: Warrior, grants: {maneuvers: 1}, training: [Heavy Armor, Heavy Shield]}
  - {name: Acolyte}
  ...
subclasses: [Paladin, Rune Knight, Paragon]
spellcasting: {model: schools, schools_chosen: 2, tag_access: [Weapon, Ward]}
paths: [Martial, Spellcaster]
```

`grants:` mirrors the ledger's `grants:` keys, so a builder can look a discipline up and know its
resource deltas without the engine special-casing it (e.g. Magus = +1 MP, +1 spell, exactly what
`tanrielle.yaml` records at L3).

## `spell_schools.yaml` (curated)

```yaml
catalog_version: 1
model: schools                 # Spellblade "draws magic from Spell Schools" (classes.md l.2834)
tag_access: [Weapon, Ward]     # a Spell with either tag is castable regardless of chosen school
schools:
  Invocation: [Close Wounds, Radiant Bolt, Luminous Burst, ...]
  Divination: [Bless, Detect Magic, ...]
# A spell is a legal pick for a Spellblade if it is in one of the character's chosen schools
# OR carries a tag in tag_access. (Primal Hide is School: Transmutation but Tags: …, Ward → legal.)
```

## `ancestries.yaml` (curated)

```yaml
catalog_version: 1
ancestries:
  Human:
    - {name: Attribute Increase, cost: 2}
    - {name: Trade Expertise, cost: 1}
    ...
  Elf:
    - {name: Nimble, cost: 2}
    - {name: Discerning Sight, cost: 0}
    ...
# cost may be negative (drawback traits refund points). Budget enforcement stays in the engine;
# the catalog only supplies the per-trait cost the engine's arithmetic checks against.
```
