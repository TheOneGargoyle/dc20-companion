DC20 Character Builder wireframe

Design pass for RUNG3_PLAN §6 (UI note). Non-functional mock. Establishes the five reusable decision widgets and the level-up-night flow before the live builder is wired to the engine. Dashed boxes are widget placeholders, not final styling.

The core idea (§6): the builder is not 40 bespoke screens. Chargen and every level-up are an ordered list of decisions the ledger + class table already dictate. So the UI is ~5 reusable widgets, and the ledger sequence decides which appear, in what order. Adding a level or a class adds data, not screens.
